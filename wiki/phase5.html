מה זה ולמה צריך את זה

Post-Processing הוא כמו עריכה בפוטושופ — אחרי שהתמונה "צולמה" (הרנדר), מוסיפים אפקטים שמשפרים את המראה הסופי. זה ההבדל בין תמונה גולמית לתמונה מעובדת.

זה בדיוק מה שמפריד בין ויואר שנראה "חובבני" לבין כזה שנראה כמו Sketchfab או Stager.

👁 SSAO — הצללה סביבתית

מוסיף צללים עדינים בפינות, חריצים, ומקומות שבהם משטחים קרובים זה לזה. כמו הצל הקל שנוצר בפינה של חדר — בלי מקור אור ספציפי, פשוט האור מגיע פחות למקומות "מוסתרים". נותן תחושה של עומק ונפח.

✨ Bloom — זוהר

כשמסתכלים על משהו בהיר מאוד (כמו שמש דרך חלון), נראה "הילה" של אור סביבו. Bloom מדמה את התופעה הזו — אזורים בהירים "מקרינים" אור לסביבתם. נותן תחושה של HDR אמיתי ומראה "קולנועי".

💪 SSR — השתקפויות

רצפה מבריקה שמשקפת את האובייקט שמעליה. SSR (Screen Space Reflections) מחשב השתקפויות ממה שכבר נראה על המסך — זול יחסית אבל מוסיף ריאליזם משמעותי.

🎨 Tone Mapping — מיפוי גוונים

מסכי מחשב מוגבלים בטווח הבהירות שהם יכולים להציג. Tone Mapping "דוחס" את כל טווח האור (מצל עמוק לשמש בוהקת) לטווח שהמסך יכול להציג, בלי לאבד פרטים. ACES הוא הסטנדרט של תעשיית הקולנוע.

📈 Anti-Aliasing — החלקת קצוות

בלי החלקה, קווים אלכסוניים נראים "משוננים" (מדרגות). Anti-Aliasing מרכך את הקצוות. FXAA מהיר אבל מטשטש קצת, TAA איכותי יותר ומשתמש במידע מפריימים קודמים.

מימוש WebGPU (שרשרת TSL)

// pipeline.ts - WebGPU Post-Processing
import { PostProcessing, pass } from 'three/webgpu';
import { bloom, ao, fxaa } from 'three/addons/tsl/display';

export function createPipeline(renderer, scene, camera) {
  const postProcessing = new PostProcessing(renderer);
  const scenePass = pass(scene, camera);

  // Get render targets
  const output = scenePass.getTextureNode('output');
  const depth = scenePass.getTextureNode('depth');

  // Chain effects via node math
  let result = output;
  result = result.add(ao(output, depth).mul(0.5));      // SSAO
  result = result.add(bloom(result).mul(0.3));          // Bloom
  result = fxaa(result);                                 // Anti-aliasing

  postProcessing.outputNode = result;
  return postProcessing;
}

💪 Stochastic SSR עם Hi-Z Ray Marching

SSR סטנדרטי מתקדם פיקסל אחר פיקסל ויכול להיות איטי. Hi-Z (Hierarchical-Z) מאיץ את התהליך על ידי שימוש ב-mip chain של מפת העומק — קפיצות גדולות כשאין פגיעה, ועידון ברזולוציה גבוהה כשיש.

Stochastic מוסיף דגימה אקראית מבוססת blue noise שמדמה את האופי הלא-מושלם של השתקפויות על חומרים מחוספסים. AMD FidelityFX SSSR הוא מימוש מוכן שאפשר ללמוד ממנו.

🎥 GTAO — Ground Truth Ambient Occlusion

גרסה מדויקת יותר של SSAO שמחשבת את ה-AO בצורה קרובה יותר למציאות. Three.js כולל מימוש של GTAO (מבוסס על Activision). הביצועים דומים ל-SSAO אבל התוצאות נקיות ומדויקות יותר.

🌐 Depth of Field (DOF) — עומק שדה

מדמה את אפקט ה"טשטוש" של מצלמה אמיתית — האובייקט בפוקוס חד, הרקע (והקידמה) מטושטשים. שימושי במיוחד להצגת מוצרים. Bokeh DOF יוצר עיגולים מטושטשים יפים.

אפשרי כאופציה שהמשתמש מפעיל — לא ברירת מחדל כי מסתיר חלק מהסצנה.

📸 Chromatic Aberration — סטיית צבע

אפקט עדין שבו הצבעים "נפרדים" קצת בקצוות התמונה — כמו שקורה בעדשות אמיתיות. ב-Three.js TSL קיים כ-chromaticAberration(). מוסיף תחושה "פוטוגרפית". יש להשתמש בעדינות!
