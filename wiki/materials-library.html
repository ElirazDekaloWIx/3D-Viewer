הסבר פשוט
מה זה חומרי PBR ולמה צריך ספרייה

חומר PBR הוא "עור" דיגיטלי שמגדיר איך משטח נראה — עץ, מתכת, בד, אבן, פלסטיק. כל חומר מורכב מ-5-6 תמונות ("מפות") שאומרות למנוע הרנדור:

  • צבע (Albedo) — הצבע הבסיסי, בלי צללים או השתקפויות
  • נורמל (Normal) — פרטי שטח מזויפים שיוצרים תחושת עומק בלי גיאומטריה נוספת
  • חספוס (Roughness) — כמה המשטח מחוספס (מראה = 0, גיר = 1)
  • מתכתיות (Metalness) — האם זה מתכת (1) או לא (0)
  • הסתרה סביבתית (AO) — צללים עדינים בחריצים ופינות
  • גובה (Displacement) — שינוי גיאומטרי אמיתי של המשטח (אופציונלי, כבד)

למה ספרייה? כדי שהמשתמש יוכל לבחור חומר (לחץ "עץ אלון" או "שיש לבן") ולראות אותו מיד על המודל שלו. כמו בחירת צבע בפוטושופ, אבל לתלת-ממד.

🔹 המלצה: ambientCG כמקור ראשי

הספרייה הגדולה ביותר (2,000+), רישיון CC0 מלא, API זמין, ומציע רזולוציות קטנות (256/512) שמושלמות לווב. Poly Haven כמשלים — מצוין לאיכות גבוהה + HDRIs.

שילוב עם Three.js

// Loading a PBR texture set
const loader = new THREE.TextureLoader();

// Load all maps
const albedo = loader.load('/materials/wood_oak/albedo.jpg');
const normal = loader.load('/materials/wood_oak/normal.jpg');
const roughness = loader.load('/materials/wood_oak/roughness.jpg');
const ao = loader.load('/materials/wood_oak/ao.jpg');

// CRITICAL: Only albedo gets sRGB color space
albedo.colorSpace = THREE.SRGBColorSpace;

// Create PBR material
const material = new THREE.MeshStandardMaterial({
  map: albedo,
  normalMap: normal,
  roughnessMap: roughness,
  aoMap: ao,
});

// Tiling
[albedo, normal, roughness, ao].forEach(tex => {
  tex.wrapS = tex.wrapT = THREE.RepeatWrapping;
  tex.repeat.set(4, 4);
});

ambientCG API — שליפת חומרים

// Fetch material list from ambientCG API
const res = await fetch(
  'https://ambientcg.com/api/v2/full_json?type=Material&limit=20&include=downloadData'
);
const data = await res.json();

// Each item has download URLs for each resolution
data.foundAssets.forEach(asset => {
  console.log(asset.assetId, asset.downloadFolders);
  // e.g. "WoodFloor051" with 1K, 2K, 4K ZIPs
});

תכנון מערכת חומרים לוויואר

🎨 רכיבי המערכת
  1. קטלוג חומרים — רשימה עם thumbnails, חיפוש וסינון לפי קטגוריה
  2. Material Loader — טוען ZIP מ-ambientCG, פורק את כל המפות, יוצר MeshStandardMaterial
  3. Material Applicator — מחיל חומר על mesh נבחר או על כל המודל
  4. Preview Sphere — כדור תצוגה מקדימה עם IBL (כמו ב-ambientCG)
  5. פרמטרים — סליידרים ל-tiling, normal strength, roughness multiplier

📦 KTX2 Texture Compression

במקום JPG/PNG שנפרסים ב-GPU ל-RGBA מלא, KTX2 (עם Basis Universal) נשאר דחוס בזיכרון ה-GPU. הקטנה של פי 10 ב-VRAM. Three.js KTX2Loader תומך בזה ישירות.

תהליך: המרה מ-JPG ל-KTX2 עם toktx (כלי של Khronos) או basisu. אפשר לעשות בצד השרת או כ-build step.

📚 MaterialX + OpenPBR

MaterialX הוא פורמט תיאור חומרים של ASWF (שמשמש את Disney, Pixar, ILM). Three.js יש MaterialXLoader אבל רק ל-WebGPU כרגע.

בעתיד, אפשר יהיה לייבא קבצי .mtlx ישירות ולקבל חומרים ברמת Stager/Blender בוויואר.

🔮 Procedural Materials (Noise-based)

במקום טקסטורות מבוססות תמונה, אפשר ליצור חומרים פרוצדורליים ב-TSL (Three Shader Language): שיש, עץ, חול, מים — הכל מחושב מתמטית ב-shader.

יתרונות: אין הורדת טקסטורות, רזולוציה אינסופית, אפשרות לאנימציה. חסרונות: עלות GPU גבוהה יותר, מוגבל לדפוסים מתמטיים.

🧰 cc0textures-threejs (npm)

ספריית קוד פתוח שטוענת חומרים מ-ambientCG ישירות ל-Three.js. CC0MaterialLoader מוריד ZIP, פורק, ומחזיר MeshStandardMaterial מוכן.
