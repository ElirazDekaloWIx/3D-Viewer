מה זה ולמה צריך את זה

כל מה שבנינו עד עכשיו (שלבים 1-6) הוא Rasterization — "ציור" מהיר שמשתמש בקיצורי דרך כדי להיראות טוב ב-60 פריימים לשנייה. זה מצוין ליומיום, אבל יש גבול לכמה זה יכול להיראות אמי��י.

Path Tracing הוא הגישה ההפוכה — סימולציה אמיתית של אור. שולחים מיליוני קרני אור וירטואליות מהמצלמה אל הסצנה, ועוקבים אחרי הנתיב שלהן (הקפצות מחומר לחומר). זה מה ש-Pixar ו-Disney משתמשים בסרטים, ומה ש-Substance Stager עושה ל"רנדר סופי".

🎥 איך זה יעבוד אצלנו
  1. ברירת מחדל: המשתמש רואה את הרנדר המהיר (שלבים 1-6) בזמן-אמת
  2. לחיצה על "איכות גבוהה" — מפעילה את Path Tracer
  3. התמונה מתחילה רועשת ומשתפרת בהדרגה (כמו תמונה שמתחדדת)
  4. אחרי 2-5 שניות — תמונה נקייה וחדה ברמה של Stager
  5. אם המשתמש מזיז את המצלמה — מתחיל מחדש (מהר)
🔮 מה זה נותן שאי-אפשר ב-Rasterization?
  • השתקפויות מרובות — מראה שמשתקפת במראה שמשתקפת ב...
  • תאורה גלובלית אמיתית — אור שמנתר מקיר אדום מאיר את הקיר הלבן ברוזי
  • קאוסטיקס — דפוסי אור שנוצרים כשאור עובר דרך זכוכית או מים
  • SSS מדויק — פיזור אור בתוך עור, שעווה, שיש
  • צללים מושלמים — רכים, מדויקים, בלי ארטיפקטים

תהליך רנדור (per frame)

// path-tracer.ts - Pseudocode for one sample
for each pixel (x, y):
  ray = generateRay(camera, x, y, randomOffset)
  color = (0, 0, 0)
  throughput = (1, 1, 1)

  for bounce = 0 to MAX_BOUNCES:
    hit = intersectBVH(ray, scene)
    if no hit:
      color += throughput * sampleEnvironment(ray.direction)
      break

    material = hit.material
    emission = material.emission
    color += throughput * emission

    // Sample BSDF for next bounce direction
    (newDirection, pdf, bsdfValue) = sampleBSDF(material, ray, hit)
    throughput *= bsdfValue / pdf

    // Russian Roulette termination
    if (bounce > 3):
      survivalProb = max(throughput.r, throughput.g, throughput.b)
      if (random() > survivalProb): break
      throughput /= survivalProb

    ray = Ray(hit.position, newDirection)

  // Blend with previous samples
  accumulated[x][y] = mix(accumulated[x][y], color, 1.0/sampleCount)

ביטול רעש (Denoising)

Monte Carlo Path Tracing מייצר רעש (דגימה אקראית). הפתרון: À-trous wavelet filter — טשטוש מבוקר שמשתמש בנורמלים ועומק כדי לשמור על קצוות חדים תוך החלקת האזורים השטוחים.

📚 WebGPU Path Tracer עם OpenPBR (ACM Web3D 2024)

מאמר אקדמי שמציג Path Tracer שרץ בדפדפן עם WebGPU Compute Shaders ומודל חומרים OpenPBR. ~10ms לדגימה, התכנסות ב-200 דגימות (2 שניות). קוד פתוח.

ArXiv: Physically-based Path Tracer using WebGPU and OpenPBR

🚀 three-gpu-pathtracer

ספריית קוד פתוח שמוסיפה Path Tracing ל-Three.js. תומכת ב-MeshPhysicalMaterial, אם כי ב-WebGL2 (לא WebGPU). מצוין כנקודת התחלה ולהבין את הארכיטקטורה.

🌈 3D Gaussian Splatting בדפדפן

טכנולוגיה חלופית ל-Path Tracing שמייצגת סצנות כ"כתמים גאוסיאניים" (splats) במקום משולשים. מאפשרת הצגת סצנות מצולמות (NeRF-like) בזמן-אמת בדפדפן.

Visionary (מעבדת AI של שנחאי) מריץ Gaussian Splatting ב-2-16ms לפריים על WebGPU, מהיר פי 135 מ-WebGL.

GitHub: Visionary  |  WebSplatter: Cross-Device Gaussian Splatting (2025)

🧠 ReSTIR — דגימה חכמה יותר

ReSTIR (Reservoir-based SpatioTemporal Importance Resampling) הוא אלגוריתם של NVIDIA שמשפר את יעילות Path Tracing על ידי שימוש חוזר בדגימות טובות ממקומות ופריימים סמוכים. מאיץ התכנסות פי 5-10 ביחס לדגימה נאיבית.

רלוונטי מאוד אם נרצה לעבור מ-200 דגימות ל-20-50 עם אותה איכות.
